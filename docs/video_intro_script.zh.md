# aieng VS Code 介绍视频脚本

## 视频定位

- **推荐时长：** 7 分钟左右
- **演示环境：** VS Code + AIENG CAD Preview 扩展 + 支持 MCP 的 Agent
- **推荐项目：** `Machined Bearing Support Bracket`
- **讲解风格：** 像给同事演示新工具，表达自然、简洁，不逐项念功能列表

## 录制布局

- 左侧打开 Agent 对话，例如 Codex、Copilot Chat 或 Claude Code。
- 右侧打开 AIENG CAD Preview。
- 底部终端仅在需要展示真实产物时短暂出现。
- CAD、仿真和优化等待过程提前录制，成片中加速或直接剪掉。

## `.aieng` 格式讲解重点

录制时不需要逐个解释所有文件，重点说明下面四件事：

1. **它是工程项目包，不是新的几何格式。**  
   STEP、STL、GLB 等真实几何仍然保存在包里；`.aieng` 负责把它们与源、语义和分析状态组织在一起。

2. **它让 Agent 能先理解项目，再执行操作。**  
   Agent 可以读取命名零件、稳定拓扑指针、可编辑参数、缺失输入和推荐步骤，不需要只靠聊天历史猜测。

3. **它保存证据和限制。**  
   求解结果、验证状态、证据索引和审计信息可以一起交付；没有可靠证据时，系统应返回未知，而不是制造结论。

4. **它支持持续迭代。**  
   CAD 修改后，下游网格或结果可以被标记为过期；项目可以被另一位工程师或另一个 Agent 接着处理。

### 三种常见项目状态

- **CAD-only：** 已经有几何、拓扑和特征信息，但还没有配置仿真。
- **CAE setup：** 已经有材料、载荷、约束或网格设置，但还没有可靠求解结果。
- **Results-backed：** 已经保存求解记录、计算指标和结果证据，可以继续做结果解释与目标比较。

这三种状态都可以是有效的 `.aieng` 项目。系统不会因为包里缺少 CAE 结果，就假装项目已经完成仿真。

### 建议展示的包结构

```text
project.aieng
├─ manifest.json              项目标识、版本与来源信息
├─ geometry/
│  ├─ source.py               Agent 生成项目中的可复现建模源
│  ├─ generated.step          Agent 生成的真实 CAD 几何
│  ├─ source.step             导入项目中的原始 STEP，可选
│  ├─ preview.glb             三维预览
│  └─ topology_map.json       B-Rep 实体与稳定面引用
├─ graph/
│  ├─ feature_graph.json      特征与可编辑参数
│  └─ constraints.json        约束与受保护区域，可选
├─ simulation/                材料、载荷、约束、网格和求解设置
├─ results/                   计算结果、摘要和证据索引
├─ validation/                验证状态、完整性与限制
├─ provenance/                工具调用与来源记录
└─ README_FOR_AI.md           面向 Agent 的项目说明，可选
```

> 实际包内成员会根据项目来源和阶段变化。没有进行 CAE 的项目可以只有 CAD 相关内容。

## 演示时使用的提示词

### 从空白项目生成模型

```text
请在当前 aieng 项目中创建一个毫米制机加工轴承支架：

- 底板尺寸为 100 × 70 × 10 mm；
- 中央立板厚 12 mm，高 60 mm；
- 立板中央创建直径 42 mm 的水平轴承孔；
- 底板四角创建直径 8 mm 的通孔；
- 添加左右对称的加强筋；
- 将零件命名为 bearing_support_bracket；
- 将主要尺寸声明为可编辑参数；
- 完成后检查模型和工程评审结果。
```

### 修改尺寸

```text
把轴承孔半径改为 23 mm，并检查这次修改有没有影响其他不相关的几何体。
```

### 使用选中的面

先在 AIENG CAD Preview 中选择底面并点击 `Copy selected`，然后发送：

```text
把选中的面设置为固定支撑。请检查当前项目是否已经具备静力分析所需的材料、
载荷、约束、网格和求解器设置。先告诉我还缺什么，不要直接运行求解器。
```

### 解释结果

```text
请读取当前项目中的真实求解结果，说明最大位移、最大应力和主要高应力区域。
如果缺少可靠结果，请直接说明，不要推测。
```

## 录制前准备

- 确认 VS Code 扩展能够连接本地 aieng 后端。
- 确认 Agent 已连接 `aieng-workbench` MCP。
- 准备一个空白项目，用于展示从自然语言生成模型。
- 准备一个带稳定面指针和可编辑参数的轴承支架项目。
- 准备一个已经完成真实求解的 CAE 项目，用于快速展示结果。
- 提前验证所有提示词，避免录制时现场调试。
- CAD 生成和求解过程在成片中加速，保留关键调用与最终结果即可。

## 可选补充镜头

如果成片还有 20 至 30 秒空间，可以快速展示：

- 导入已有 STEP 或打开本地 `.aieng` 包。
- 给真实产品附加参考图片，辅助 Agent 校准比例。
- 拓扑优化结果写回模型。
- 修改 CAD 后，旧网格和旧结果被标记为需要重新验证。

## 表述边界

- 可以说：真实 CAD、真实导出、真实求解证据、可编辑参数、稳定几何指针。
- 不要说：自动保证设计正确、生产认证、无需工程师审核。
- 展示合成热图时，必须明确说明它不是真实求解结果。
