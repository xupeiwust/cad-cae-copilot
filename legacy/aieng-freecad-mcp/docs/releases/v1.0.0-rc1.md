# Release Notes — v1.0.0-rc1

**Release date:** 2026-05-13  
**Status:** Release Candidate

## Summary

v1.0.0-rc1 hardens the composable architecture introduced in v0.9.x and adds a
read-only capability inspection tool, CI integration, and release-quality documentation.
This is a **release candidate** intended for integrator testing before the
final v1.0.0.

## What's New

### Composable Architecture
- Reframed CAD and CAE as **independent, first-class workflows**.
- Five supported paths:
  1. CAD-only (inspect, modify, export)
  2. CAE-only (setup, mesh, solve, postprocess)
  3. Optional CAD -> CAE (explicit orchestration)
  4. Reference (traceability maps)
  5. Claim (explicit claim update with evidence)
- Added optional orchestration helper:
  - `aieng_run_cad_to_cae_workflow`
  - Alias: `aieng_orchestrate_cad_cae_sequence`

### Capability Inspection Tool
- New module: `src/freecad_mcp/aieng_bridge/planner.py`
- New MCP tool: `aieng_inspect_capabilities`
- Alias: `aieng_plan_capabilities` (deprecated, backward-compatible)
- Read-only, planning-neutral capability inspection that exposes:
  - Possibly relevant tools (by category, unordered, unranked)
  - Package context and runtime capabilities
  - Missing information
  - Unsupported operations
  - Side effects and policy reminders
- Never executes operations, modifies packages, or advances claims.
- Never prescribes workflow sequences or ranking.
- Returns `CapabilityInspectionSummary` with:
  - `possibly_relevant_tools`
  - `missing_information`
  - `unsupported_operations`
  - `needs_review`
  - `policy_reminders`

### Audit and Reporting
- `aieng_generate_audit_report` MCP tool
- Detects claim discipline violations:
  - `claims_advanced=True` in non-claim-update evidence
  - Non-`aieng_update_claim` claim map modifications
- Outputs `reports/audit_report.md` and `.json`

### CI / CD
- Added `.github/workflows/tests.yml`
  - Runs on Python 3.11, 3.12, 3.13
  - No FreeCAD required (mock/surrogate mode)
  - Triggered on push/PR to `main`/`master`

### Documentation
- `docs/known_limitations.md` — explicit limitation inventory
- `docs/example_v1_demo_output.md` — representative demo output
- `README.md` updated for composable architecture emphasis
- `docs/tool_contract.md` updated with planner tool schema

## Known Limitations

See `docs/known_limitations.md` for the full list. Key items:

- Topology edits unsupported (face/edge ID stability not guaranteed)
- Linear static structural CAE only
- No automatic claim advancement
- Compound claim logic not implemented
- FreeCAD is optional; real execution requires FreeCAD + FEM workbench

## Migration from v0.9.x

No breaking changes. All v0.9.x tools and APIs remain compatible.
The `aieng_run_cad_to_cae_workflow` tool has been clarified as an **optional
explicit orchestration helper**, not an automatic pipeline.
