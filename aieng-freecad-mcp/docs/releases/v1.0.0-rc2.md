# Release Notes — v1.0.0-rc2

**Release date:** 2026-05-13  
**Status:** Release Candidate 2

## Summary

v1.0.0-rc2 closes the Milestone 1 acceptance gate and hardens the error taxonomy, contract consistency, and CI reliability. All 11 Milestone 1 acceptance criteria pass under `--dry-run`. The release candidate remains at `1.0.0rc1` in `pyproject.toml` pending final promotion.

## What's New Since rc1

### Milestone 1 Acceptance Gate
- Added `scripts/run_milestone1_acceptance.py` — repeatable driver for the 11 AGENTS.md acceptance criteria.
- Added `--dry-run` mode that stubs FreeCAD calls while exercising real file I/O, patch parsing, guard checks, evidence/trace persistence, and claim discipline.
- Added `tests/test_milestone1_acceptance_dryrun.py` — subprocess test asserting 11/11 PASS in dry-run mode.
- Created minimal stub FCStd fixture (`examples/parametric_bracket/freecad/source.FCStd`) for existence checks without FreeCAD.

### Error Taxonomy & Contract Consistency
- Added unified failure taxonomy in `src/freecad_mcp/contracts/failure_mode.py`:
  - `FailureMode` str subclass with 14 canonical values.
  - `FailureDetail` Pydantic model.
  - `classify_exception()` heuristic.
- Added 21 error code constants (e.g., `INVALID_INPUT`, `MISSING_ARTIFACT`, `PERSISTENCE_FAILED`, `POLICY_VIOLATION`).
- Added unified mapping layer:
  - `map_failure_mode_to_error_code()`
  - `derive_primary_error_code()`
- Auto-derived `primary_error_code` on `StandardToolResult`, `CadToolResponse`, `CaeBaseResponse` via Pydantic validators.
- `CaeErrorResponse` falls back from legacy `error_code` to mapped canonical code.
- Added `scripts/validate_error_code_consistency.py` — read-only scanner comparing code constants ↔ doc tables ↔ schema fields.
- Wired consistency check into `.github/workflows/tests.yml`.

### CI Hardening
- Bumped `actions/checkout` to v5 and `actions/setup-python` to v6.
- Set `PYTHONPATH` at job level.
- Added health check step verifying `import freecad_mcp`, `import yaml`, and key package versions.
- Added error-code consistency validation step.

### CI Follow-up for Phase 1 Authoring Pipeline
- Fixed the default GitHub Actions test workflow after the Phase 1 authoring pipeline backend landed on `main`.
- Root cause: the first CI fix attempted to install the private sibling `aieng` repository via `pip install git+https://github.com/armpro24-blip/aieng.git`, which failed under GitHub Actions because the repository is not anonymously cloneable.
- Removed the private cross-repo install from the default workflow and from dev dependencies.
- Added a local fallback for `BackendExecutionResult` and `StepExecutionResult` in `src/freecad_mcp/aieng_bridge/modeling_executor.py` so standalone CI can import the backend module without the sibling `aieng` package installed.
- Replaced the test that depended on `aieng.backend_discovery` with a local entry-point registration check for the `aieng.backends` group.
- Verified the repaired `Tests` workflow passed on commit `030e291` (run `25916246051`).

### Documentation
- Updated `docs/acceptance_tests.md` with dry-run mode documentation.
- Updated `docs/known_limitations.md` last-updated tag (v1.0.0-rc1 — still current).

## Out of Scope for Milestone 1

These items are explicitly outside the Milestone 1 boundary and remain planned for future versions:

- **Real FreeCAD execution in CI** — Milestone 1 acceptance is gated on `--dry-run` because FreeCAD is not available in CI. Real execution remains environment-dependent.
- **Topology-changing CAD edits** — Only parametric regeneration with executable edit metadata is supported.
- **Nonlinear/contact/buckling/fatigue/CFD/thermal CAE** — Linear static structural only.
- **Design optimization** — No topology, shape, or sizing loops.
- **Compound claim logic** — AND/OR/NOT across claims is not implemented.
- **Automatic claim advancement** — Claim updates remain explicit via `aieng_update_claim` only.

## Known Limitations

See `docs/known_limitations.md` for the full inventory. Key items:

- Topology edits unsupported (face/edge ID stability not guaranteed).
- Linear static structural CAE only.
- No automatic claim advancement.
- Compound claim logic not implemented.
- FreeCAD is optional; real execution requires FreeCAD + FEM workbench.
- Windows path handling: internal paths use forward slashes; some FreeCAD external tools may expect native Windows paths.
