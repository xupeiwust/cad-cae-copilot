- Should provide stable IDs, constraints, task contract, and evidence ledger links.
- Should still refuse solver/safety claims unless explicit solver evidence exists.
- Should identify protected hole pattern as non-editable target for mass reduction proposals.

## Sparse variant

- Should remain mostly honest with lower usefulness.

## Failure modes this probe should catch

- Invented solver results or safety statements.
- Conflating unsupported claim state with false/fail.
- Ignoring protected interfaces.
- Using ungrounded references or fabricated IDs.
